## 제품 기회 찾기

최고의 AI 제품은 실제 문제를 해결합니다. 추구할 가치가 있는 아이디어를 발견하는 방법을 알아보세요.

**핵심 원칙:** 기술이 아닌 문제부터 시작하세요. AI는 그 자체로 해결책이 아닌 도구입니다.

### 아이디어를 찾을 곳

1. **자신의 불편사항**
   - 어떤 반복적인 작업이 짜증나나요?
   - 워크플로우에서 무엇이 너무 오래 걸리나요?
   - 어떤 전문 지식이 필요하지만 없나요?

   예: "매일 고객 이메일 분류에 2시간 소요" → AI 이메일 분류기

2. **산업 문제**
   - 특정 산업의 전문가와 대화
   - 워크플로우 비효율성 관찰
   - 병목 현상 식별

   예: 변호사들이 문서 검토에 10시간 이상 소요 → AI 법률 문서 분석기

3. **새로운 트렌드**
   - 새로운 AI 기능 (GPT-4, Claude 3 등)
   - 규제 변화
   - 시장 변화

   예: GDPR 준수 → AI 개인정보 처리방침 생성기

## 아이디어 발굴 프로세스

### 1단계: 문제 식별

**방법 1: 사용자 인터뷰**
```
질문 목록:
- 업무에서 가장 시간이 많이 걸리는 부분은?
- 자동화하고 싶은 작업은?
- 어떤 전문 지식이 필요하지만 부족한가?
- 반복적으로 하는 실수는?
- 주당 몇 시간을 절약할 수 있다면 좋겠는가?
```

**인터뷰 결과 예시:**
```
사용자: 소규모 사업체 소유자
불편사항: "제품 설명 작성에 주 5시간 낭비"
빈도: 매주
지불 의향: 월 $50
→ 잠재 제품: AI 제품 설명 생성기
```

**방법 2: 시장 조사**
```python
market_research = {
    "대상 고객": "누가 이 문제를 가지고 있나?",
    "시장 규모": "잠재 사용자가 몇 명인가?",
    "현재 솔루션": "이미 존재하는 것은?",
    "공백": "무엇이 빠져 있나?",
    "예산": "현재 무엇에 지출하나?"
}
```

**실제 예시:**
- 대상: 전자상거래 판매자 (전 세계 5천만 이상)
- 문제: 제품 설명 작성
- 현재 솔루션: 카피라이터 (설명당 $100-500)
- 공백: 저렴하고 빠른 고품질 대안
- 예산: 월 $200-2000 카피라이팅 비용

### 2단계: AI 적합성 평가

**AI가 적합한 경우:**

✅ **적합:**
- 패턴이 있는 반복 작업
- 대량 데이터 분석 필요 작업
- 콘텐츠 생성
- 분류/범주화
- 과거 데이터 기반 예측
- 언어 이해/생성

❌ **부적합:**
- 물리적 존재 필요 작업
- 생명 중요 결정 (감독 없는 의료 진단)
- 독특한 인간 관점 필요한 창작 작업
- 데이터 부족 작업
- 적절한 감독 없는 고도 규제 영역

**기술적 타당성 체크리스트:**

```python
feasibility_check = {
    "데이터 가용성": "✓ 학습 데이터를 얻거나 기존 모델 사용 가능?",
    "API 가용성": "✓ GPT-4, Claude 등 사용 가능?",
    "지연 시간 허용": "✓ 사용자가 1-10초 대기 가능?",
    "정확도 요구사항": "✓ 80-95% 정확도로 충분?",
    "비용 지속가능성": "✓ API 비용 < 수익의 30%?"
}
```

**평가 예시:**
```
아이디어: AI 이력서 스크리닝
✓ 데이터: 가능 (공개 이력서 데이터셋)
✓ API: GPT-4로 이력서 분석 가능
✓ 지연시간: 5초 허용 가능
✓ 정확도: 90%면 유용 (사람이 여전히 검토)
✓ 비용: 이력서당 $0.01-0.05 (이력서당 $0.50 가격책정 시 지속가능)
→ 실현 가능
```

### 3단계: 검증 방법

**1. 랜딩 페이지 테스트**
```html
<!-- 간단한 랜딩 페이지 생성 -->
<h1>AI 이메일 분류기 - 매일 2시간 절약</h1>
<p>AI로 이메일을 자동으로 정렬하고 우선순위 지정</p>
<form>
  <input placeholder="얼리 액세스를 위한 이메일 입력">
  <button>얼리 액세스 받기</button>
</form>

<!-- $50-200 광고 집행 -->
<!-- 좋은 신호: 5-10% 전환율 -->
```

**측정 지표:**
- 페이지 방문: 1000
- 가입: 80 (8% 전환율)
- 지불 의향 추정: 가입 후 설문조사
→ 50% 이상이 월 $10-20 지불 의향 있으면 유망

**2. 수동 MVP (컨시어지 테스트)**

자동화를 구축하기 전에 수동으로 서비스 제공:

```
예: AI 기반 콘텐츠 생성

1주차:
- 5명의 베타 테스터 확보
- 이메일로 요청 제출
- 직접 콘텐츠 작성 (AI 시뮬레이션)
- 24시간 내 전달

학습 내용:
- 사용자가 기대하는 품질은?
- 허용 가능한 처리 시간은?
- 얼마를 지불할 의향이 있나?
- 어떤 기능이 가장 중요한가?

만족하고 지불 의향 있으면 → AI 버전 구축
```

## 평가 프레임워크

### 1. 영향력 vs 노력 매트릭스

```
높은 영향, 낮은 노력 (먼저)  | 높은 영향, 높은 노력 (계획)
------------------------------|---------------------------
- 이메일 분류기              | - AI가 있는 완전한 CRM
- 이력서 스크리너            | - AI 의료 진단
- 소셜 미디어 캡션 생성기    | - 자율주행 차량 소프트웨어

낮은 영향, 낮은 노력 (고려)  | 낮은 영향, 높은 노력 (회피)
------------------------------|---------------------------
- 간단한 챗봇                | - 복잡한 기업용 소프트웨어
- 기본 텍스트 요약기          | - 완전 맞춤형 LLM 학습
```

**점수 시스템:**
```python
def score_idea(problem, solution):
    impact = {
        "영향받는_사용자": 0-10,     # 얼마나 많은 사람이 이 문제를 가지고 있나?
        "고통_강도": 0-10,           # 문제가 얼마나 고통스러운가?
        "빈도": 0-10,                # 얼마나 자주 발생하나?
        "지불_의향": 0-10            # 솔루션에 돈을 낼까?
    }
    
    effort = {
        "기술_복잡성": 0-10,          # 구축하기 얼마나 어려운가?
        "시장_출시_시간": 0-10,       # 출시까지 얼마나 걸리나?
        "리소스_요구사항": 0-10       # 필요한 돈/팀은?
    }
    
    impact_score = sum(impact.values()) / 4
    effort_score = sum(effort.values()) / 3
    
    priority = impact_score / effort_score
    return priority  # 높을수록 좋음
```

## 사례 연구

### 사례 1: NotionAI

**발견 과정:**
- 문제: Notion에서 작성 및 편집에 시간 소요
- 사용자: 3천만 이상의 Notion 사용자
- 검증: Notion 커뮤니티의 높은 수요
- 솔루션: 내장 AI 작성 도우미
- 결과: 매우 성공적인 애드온

**핵심 학습:** 기존 워크플로우에 AI 통합 (사용자가 행동을 바꾸도록 요구하지 말 것)

### 사례 2: Jasper AI

**발견 과정:**
- 문제: 마케팅 팀이 빠른 콘텐츠 필요
- 시장 조사: $4000억 콘텐츠 마케팅 산업
- 검증: 100명 이상의 베타 테스터가 월 $50 지불
- 구축: AI 카피라이팅 도구
- 결과: 18개월 만에 $1억 2500만 수익

**핵심 학습:** 명확한 ROI가 있는 고가치 문제 타겟

### 사례 3: Grammarly

**발견 과정:**
- 문제: 모든 사람이 작문 실수를 함
- 거대한 시장: 글을 쓰는 모든 사람
- 시작: 간단한 문법 검사기
- 진화: AI 기반 작문 도우미
- 결과: 일 3천만 이상 사용자

**핵심 학습:** 단순하게 시작하고 나중에 AI로 확장

## 실습: 아이디어 찾기

**1-2일차: 브레인스토밍 (50개 아이디어)**
```
1. 자신의 불편사항 나열 (10개)
2. 5명과 워크플로우에 대해 인터뷰 (각 10개 불편사항)
3. r/entrepreneur + 산업 포럼 탐색 (20개 불만 수집)
→ 총: 50개 이상의 잠재 아이디어
```

**3-4일차: 필터링 (10개 아이디어)**
```
필터 적용:
- AI가 실제로 도움이 될까? (예/아니오)
- 사람들이 적극적으로 솔루션을 찾고 있나? (Google Trends)
- 이것을 위한 예산이 있나? (>월 $100 문제)
→ 10개 아이디어로 좁히기
```

**5-7일차: 심층 검증 (3개 아이디어)**
```
상위 3개 아이디어에 대해:
- 랜딩 페이지 생성
- $50 광고 집행
- 잠재 고객 10명 인터뷰
- 경쟁 조사
→ 최고의 것 선택!
```
