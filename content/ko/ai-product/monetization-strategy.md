## AI 제품의 수익 모델

**핵심 원칙:** 비용이 아닌 가치를 기반으로 가격을 책정하세요. 사용자의 주당 10시간을 절약한다면, API 비용이 아닌 그 가치를 기준으로 요금을 부과하세요.

## 가격 전략

### 1. 구독 기반 (가장 일반적)

**구조:**
```python
pricing = {
    "무료": {
        "가격": 0,
        "기능": ["월 10회 생성", "기본 AI"],
        "목표": "사용자 확보, 가치 증명"
    },
    "Pro": {
        "가격": 29,
        "기능": ["월 500회 생성", "GPT-4", "우선순위"],
        "목표": "주요 수익원"
    },
    "비즈니스": {
        "가격": 99,
        "기능": ["무제한", "API 액세스", "화이트레이블"],
        "목표": "고가치 고객"
    }
}
```

**실제 예시 - Jasper AI:**
- 무료: 10,000 단어
- Creator ($49/월): 50,000 단어
- Teams ($125/월): 150,000 단어
- 비즈니스: 맞춤 가격

**사용 시기:** 예측 가능한 수익, 고객이 예산 계획하기 쉬움

### 2. 프리미엄 모델 (성장에 최적)

**무료 티어 전략:**
```
충분히 제공:
✓ 핵심 가치 경험
✓ 습관 형성
✓ 가능성 확인

하지만 제한:
✗ 사용량 (월 10회 vs 500회)
✗ 속도 (일반 vs 우선순위 대기열)
✗ 기능 (기본 vs 고급 AI)
✗ 지원 (커뮤니티 vs 이메일)
```

**전환 퍼널:**
```
1000명 가입 (무료)
  ↓ 30% 활성화
300명 활성 사용자
  ↓ 10% 전환
30명 유료 사용자 ($29/월)
  = $870 MRR
```

**실제 예시 - Grammarly:**
- 무료: 기본 문법 검사
- 프리미엄 ($12/월): 고급 제안, 표절 검사
- 비즈니스 ($15/사용자/월): 팀 기능
- 전환율: ~5% 무료 → 유료

### 3. 사용량 기반 결제

**최적:** API, 사용량 편차가 큰 경우

**가격 구조:**
```python
api_pricing = {
    "요청당": 0.01,  # 요청당 $0.01
    "계층": {
        "0-1000": 0.01,      # 첫 1K 요청
        "1001-10000": 0.008,  # 다음 9K (20% 할인)
        "10001+": 0.005       # 10K+ (50% 할인)
    }
}

# 청구서 예시:
# 고객이 월 15,000회 요청
# 1,000 × $0.01 = $10
# 9,000 × $0.008 = $72
# 5,000 × $0.005 = $25
# 합계: $107
```

**실제 예시 - OpenAI:**
- GPT-4: 입력 토큰 1K당 $0.03
- GPT-3.5: 토큰 1K당 $0.002
- Whisper: 분당 $0.006
- DALL-E: 이미지당 $0.04

### 5. 기업 라이선스

**100명 이상 기업용:**
```
엔터프라이즈 패키지:
- 맞춤 가격 ($500-5000/월)
- 전담 지원
- SLA 보장 (99.9% 가동 시간)
- 맞춤 통합
- 화이트레이블 옵션
- 교육 세션
- 우선 기능
```

**영업 프로세스:**
1. 인바운드 리드 (무료/프로 티어에서)
2. 발견 통화
3. 맞춤 제안
4. 협상
5. 계약 (연간)

## 가격 고려사항

### 비용 분석

**단위 경제성 계산:**
```python
def calculate_margins(product="blog_post"):
    # 생성당 비용
    costs = {
        "gpt4_api": 0.15,      # API 호출
        "호스팅": 0.02,         # 서버 비용
        "처리": 0.01,           # 컴퓨팅
        "결제_수수료": 0.09     # Stripe (3%)
    }
    
    total_cost = sum(costs.values())  # $0.27
    
    price = 3.00  # 블로그 게시물당 $3 청구
    margin = price - total_cost  # $2.73 (91%)
    
    return {
        "비용": total_cost,
        "가격": price,
        "마진": margin,
        "마진_퍼센트": (margin/price) * 100
    }

# 목표: SaaS의 경우 60-80% 마진
```

**월간 비용 예측:**
```
사용자: 100명
평균 사용량: 사용자당 월 50회 생성
총계: 5,000회 생성

비용:
- API: 5,000 × $0.15 = $750
- 호스팅: $50
- 결제 처리: $90
- 합계: $890

수익 (100명 × $29): $2,900
이익: $2,010 (69% 마진)
```

## 성공 사례

### 사례 1: Midjourney
**모델:** 구독 + 사용량 제한
- 기본 ($10/월): 200개 이미지
- 표준 ($30/월): 무제한 느린 모드, 15시간 빠른 모드
- Pro ($60/월): 무제한 + 스텔스 모드
- Mega ($120/월): 60시간 빠른 모드

**수익:** 간단한 가격으로 연 $2억 수익

### 사례 2: Notion AI
**모델:** 기존 제품에 추가
- 기본 Notion: 무료/$8/$15 사용자당
- + AI: 사용자당 월 $10 추가

**전략:** 기존 3천만 사용자 활용

### 사례 3: GitHub Copilot
**모델:** 정액 구독
- 개인: 월 $10 또는 연 $100
- 비즈니스: 사용자당 월 $19

**성공:** 첫 해 100만 이상 유료 사용자

## 구현 가이드

### 1단계: 출시 가격 (1-3개월)
```
간단하게 시작:
- 무료: 제한된 사용량
- Pro ($29/월): 주요 티어
- 선택사항: 연간 할인 (20% 할인)

목표:
- 100명의 유료 고객 확보
- 사용 패턴 학습
- 피드백 수집
```

### 2단계: 최적화 (4-6개월)
```
데이터 기반 추가:
- 격차가 있으면 새 티어
- 편차가 크면 사용량 기반
- 대기업이 문의하면 엔터프라이즈

A/B 가격 테스트:
- $29 vs $39 시도
- 전환율 측정
```

### 3단계: 확장 (7개월+)
```
개선:
- 연간 플랜 (더 나은 유지율)
- 추가 기능 (별도 요금)
- 볼륨 할인
- 파트너/제휴 가격
```

## 일반적인 실수

### 실수 1: 가격이 너무 낮음
❌ "사용자를 유치하기 위해 월 $5"
✅ "월 $29 - 우리는 $500 가치 제공"

### 실수 2: 너무 많은 티어
❌ 7개 가격 티어
✅ 최대 2-3개 티어

### 실수 3: 가격 숨기기
❌ "가격은 문의하세요"
✅ "웹사이트에 명확한 가격"

### 실수 4: 테스트 안 함
❌ "가격 설정 후 방치"
✅ "분기별 가격 A/B 테스트"
